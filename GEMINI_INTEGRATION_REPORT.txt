                            RESUMO EXECUTIVO
================================================================================

O objetivo geral foi integrar a API do Google Gemini ao aplicativo Expo React 
Native para:
  • Calcular IMC (Índice de Massa Corporal) dos usuários
  • Gerar recomendações personalizadas de dieta
  • Proporcionar uma experiência suave mesmo sob alta demanda da API

Resultado: Implementação bem-sucedida com retry automático e exponential backoff
para tratamento de erros 503/429.


================================================================================
                        HISTÓRICO DE PROBLEMAS
================================================================================

PROBLEMA #1: "An API Key must be set when running in a browser"
────────────────────────────────────────────────────────────────────────────
Causa: Uso da biblioteca @google/generative-ai (GoogleGenerativeAI) no 
       navegador/Expo Web sem configuração adequada de API key.
       
Sintoma: Erro crítico ao tentar instanciar GoogleGenerativeAI({}) no app/result.tsx

Solução: 
  • Remover TODAS as importações da biblioteca @google/generative-ai
  • Abandonar SDK do Google em prol de fetch nativo (puro JavaScript)
  • Usar a API REST do Gemini via fetch() com headers corretos


PROBLEMA #2: "Erro 404 - Model not found"
────────────────────────────────────────────────────────────────────────────
Causa: Versão incorreta do modelo na URL da API
       URL usada: https://.../models/gemini-1.5-flash-latest:generateContent?key=...
       
Sintoma: API retornava 404 mesmo com API key válida

Solução:
  • Mudar modelo para: gemini-flash-latest
  • Remover parâmetro de versão explícita ?key= da URL
  • Usar header X-goog-api-key em vez de query parameter


PROBLEMA #3: "Erro 400 - Bad Request"
────────────────────────────────────────────────────────────────────────────
Possíveis causas identificadas:

  a) Formato do Payload (Body) incorreto
     - Estava enviando: { sexo: 'Masculino', idade: 20, ... } direto no text
     - API esperava: { contents: [{ parts: [{ text: "mensagem montada" }] }] }
     
  b) Valores quebrando o JSON
     - Altura com vírgula (ex: '1,75') corompendo a estrutura JSON
     - Falta de validação dos dados
     
  c) Headers obrigatórios faltando
     - Content-Type: application/json não estava sendo enviado
     
  d) Método HTTP errado
     - GET em vez de POST (não permitido com body)

Solução: Refatoração completa do payload, validação e headers


PROBLEMA #4: "Erro 503 - Service Unavailable / 429 - Too Many Requests"
────────────────────────────────────────────────────────────────────────────
Causa: Alta demanda na API do Gemini durante períodos de pico

Sintoma: Requisição falha ocasionalmente, quebrando a experiência do usuário

Solução: Implementação de retry automático com exponential backoff

FASE 3: ADIÇÃO DE RETRY COM EXPONENTIAL BACKOFF (Solução final)
───────────────────────────────────────────────────────────────
Implementado dentro de requestGemini():
  
  Loop com até 3 tentativas:
    • Tentativa 1: se falha com 503/429 → espera 1 segundo
    • Tentativa 2: se falha com 503/429 → espera 2 segundos  
    • Tentativa 3: se falha com 503/429 → espera 4 segundos
    • Após 3 falhas → throw error para tratamento no front-end

Fórmula do backoff: Math.pow(2, attempt - 1) * 1000 ms
  • 2^0 * 1000 = 1000ms (1s)
  • 2^1 * 1000 = 2000ms (2s)
  • 2^2 * 1000 = 4000ms (4s)

Logs adicionados por tentativa:
  • console.log(`[Gemini] Attempt X/3 - Sending...`)
  • console.log(`[Gemini] Attempt X - Response status: ...`)
  • console.warn(`[Gemini] Attempt X returned 503. Retrying in 1000ms...`)
  • console.error(`[Gemini] Attempt X failed: ...`)


Estrutura final de requestGemini():
┌─ for loop (attempt 1 a 3)
│  ├─ try block
│  │  ├─ fetch POST com headers corretos
│  │  ├─ check se status é 503 ou 429
│  │  │  ├─ Se sim e tentativas restantes: wait + continue
│  │  │  ├─ Se não e tentativas restantes: continue loop normal
│  │  │  └─ Se não: parse JSON e retorna texto
│  │  └─ retorna data?.candidates[0]?.content?.parts[0]?.text
│  └─ catch block
│     ├─ grava erro em lastError
│     ├─ verifica se deve retentar
│     └─ throw ou continue para próxima tentativa
└─ fallback (nunca deve chegar aqui)
